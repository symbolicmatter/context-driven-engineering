Context-Driven Engineering

Context-Driven Engineering (CDE) treats context as the primary engineering artifact and code as a derived outcome, especially in AI-assisted development. The goal is simple: make design intent explicit enough that both humans and AI agents can work reliably.

CDE focuses on:

  • Explicit design intent (vision, specs, architecture, constraints)
  • Coherent documentation that is usable as execution context
  • Reliable collaboration between humans and AI coding agents
Reference repo on GitHub

CDE is intentionally worldview-agnostic. The reference repository includes an Object-Centric Design Worldview as a worked example of an explicit design worldview, plus a set of practical agent rules in AGENTS.md.

If you’re looking for a framework, this probably isn’t it. If you’re looking for a way to make AI-assisted engineering safer and more coherent, you’re in the right place.
